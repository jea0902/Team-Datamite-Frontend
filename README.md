### [프론트엔드 핵심 스킬]()
(상세하게 보시려면 프론트엔드 핵심 스킬 링크를 클릭해주세요)

#### 메인기능
- 챗봇 커스텀 디자인 사용
- 카카오 지도 API 사용 및 커스텀 디자인 사용
- 이미지 분석 페이지
- 태블로를 사용한 분석 페이지
- 사용자 친화적이고, 세련된 UI 디자인

#### 기본 기능
- 회원 관리 기능(회원가입, 로그인, 로그아웃)
- 마이페이지 기능

<br /><br />

### 서비스 기획 배경
- 현실에서 많은 사람들이 건강 상의 문제로 어떤 병원을 찾아야 하는지 혼란을 겪는 상황을 개선하고자 하는 의도에서 시작했습니다. 증상이 발생하면 어떤 진료과에 해당하는지, 해당 진료과로 어떤 병원을 가야할지 등의 정보를 얻기 위해서는 여러 단계의 검색과 의사결정 과정이 필요합니다. 이로 인해 환자들은 시간 낭비 등과 같은 불편을 경험하게 되며, 때로는 적절한 의료 서비스를 받지 못할 수도 있습니다.
- 기존 대부분의 의료 정보 제공 서비스들은 비대면 진료와 병원 예약에 치중되어 있었고, 진료과 추천 서비스의 경우 반복적인 질문과 한계가 있는 선택지로 증상을 입력받습니다.
- 이를 개선하기 위해 자연어 처리 기술을 활용하여 사용자가 증상을 편리하게 입력할 수 있도록 하고, 그에 따라 가장 적절한 진료과와 관련 정보를 제공함으로써 환자들의 의료 서비스 접근성을 높일 수 있다고 생각했습니다.

<br />

오약
- 환자가 어느 진료과에 갈지,
- 내 증상이 어떤 진단명과 유사한지,
- 근처 병원은 어디에 있고,
- 그 병원의 리뷰는 어떤지를 <br />

여러번의 서치없이 한번에 정보를 제공해주는 서비스의 필요성을 느끼고,<br /> 한 페이지에서 쉽고 빠르게 병원 내원을 돕기 위해 제작

<br /><br />

### 프로젝트 주제
- 환자의 증상별 진료과목 및 진료 병원 정보 제공 서비스
- 자연어 처리 기반 종합 의료 정보 제공 웹 서비스

<br /><br />

### 프로젝트 목표
- 환자들이 빠르고 정확한 의료 정보를 쉽게 얻을 수 있도록 하며, 의료 서비스에 대한 투명하고, 신뢰성 있는 정보를 제공하는 걸 목표로 합니다.

<br /><br />

### 프로젝트 핵심 요약

세줄 요약 : <br />
 1. 사용자가 증상만 텍스트로 챗봇에 입력하면 ➝ 진료과, 진단명, 주변 병원, 병원 리뷰를 한 페이지에서 제공
 2. 피부 사진을 찍어 업로드하면 ➝ 진단 결과를 제공
 3. 태블로로 회원들의 다양한 데이터를 활용 ➝ 분석 내용을 시각화로 제공

<br /><br />

### 주요 기능
- 챗봇에 증상 입력으로 12개 진료과 분류
- 환자의 증상과 유사한 질환명 제공
- 현재 위치 기반 해당 진료과 병원 정보 제공
- 병원 리뷰 요약 정보 제공
- 피부 질환 이미지 분류
- 실시간 의료 정보를 반영한 대시보드 제공

<br /><br />

### 주요 기능들을 풀어 설명

#### 챗봇 기능
- 사용자가 증상만 자연어로 챗봇에 입력하면 ➝ 그 증상에 대한 진료과와 예상 진단명을 매칭해주고, 진료과에 해당하는 주변 병원들을 카카오 지도로 정보를 제공합니다.
- 각 병원들을 클릭하면 그 병원의 리뷰와 리뷰 분석을 통한 ➝  리뷰 키워드를 비롯해 거리, 전화번호, 진료시간 등 기본정보도 같이 보여줍니다.
- 이 모든 기능이 한 페이지 안에서 짧은 시간 안에 이루어지기에 사용자 친화적입니다.
#### 이미지 분석 기능
- 사용자가 피부 사진을 찍어 업로드하면 ➝ 사진을 특정 피부질환에 분류하여 진단 결과를 제공합니다.
#### 마이페이지 기능
- 회원가입을 한 회원들은 챗봇을 사용한 이력을 저장하고, 채팅 기록을 당시 챗봇 UI 그대로 볼 수 있습니다.
- 태블로를 활용해 대시보드 페이지를 만들어, 질문 건수와 회원 수, 성비, 연도별 질문 건수, 진료과 별 질문 건수, 계절별 진료과 수, 전국지도 지역별 진료과 비율, 리뷰 긍/부정 비율 등 다양한 정보를 한 페이지에서 제공합니다.

<br /><br />
